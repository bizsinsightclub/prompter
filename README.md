# PDF 텔레프롬프터

발표 **PDF**를 불러와 슬라이드로 쓰고, 페이지별 **대본**을 브라우저에서 편집하며, **라이브 싱크·폰 리모컨·영상 삽입**이 기본 내장된 정적 웹 앱. 서버 백엔드·빌드 도구 없음. 기본 상태는 비어 있고 발표 콘텐츠는 실행 시 PDF로 직접 불러온다.

🔗 **라이브:** https://bizsinsightclub.github.io/prompter/

---

## 시작 화면 — PDF 불러오기

처음 열면 PDF 입력 화면이 뜬다. 슬라이드 PDF를 드래그&드롭(또는 클릭 선택)하면 각 페이지를 이미지로 변환해 페이지별로 편성한다. 불러온 PDF는 이 브라우저에 저장되어 다음에 자동으로 다시 열린다. 여기서 **총 발표 시간(분)**도 설정한다(이후 타이머 영역에서 언제든 변경 가능).

![시작 화면](docs/start.png)

## 프롬프터(호스트) 화면

발표자가 보는 메인 화면. 왼쪽 사이드바에 현재 페이지 썸네일·**마운트된 PDF 파일명**·큰 타이머·**페이지별 권장시간(설정 가능)**·영상 첨부, 오른쪽에 대본과 **워드형 서식 툴바**(굵게·기울임·밑줄·글자 크기·**글자색(흰색/강조/빨강/파랑/초록/직접 선택)**·목록·서식 지우기).

![프롬프터 편집 화면](docs/prompter.png)

## 기기 연결 — 폰 리모컨 / 연단 슬라이드

**🔗 기기 연결**을 누르면 3자리 코드와 폰 QR이 뜬다. 폰은 QR 스캔으로 즉시 연결(입력 불필요), 다른 PC의 연단 슬라이드는 `deck.html`을 열고 3자리 코드를 입력해 연결한다.

![기기 연결 패널](docs/connect.png)

## 연단 슬라이드(deck)

프로젝터에 띄우는 전체화면 슬라이드. **이 기기엔 PDF가 필요 없다** — 프롬프터가 현재 페이지 이미지를 직접 전송한다(2560px 고화질).

![연단 슬라이드](docs/deck.png)

## 실행

정적 서버 또는 위 GitHub Pages URL로 연다. (`file://` 직접 열기는 IndexedDB·WebRTC가 불안정)

```bash
python -m http.server 8080      # 또는 npx serve -l 8080 .
# → http://localhost:8080/  (index.html → prompter.html 로 이동)
```

## 핵심 기능

### 1. 대본 입력
- **개별 편집** — `E` 또는 ✎ 편집 → 워드형 툴바로 서식 지정. 입력 즉시 자동 저장. 빈 페이지는 placeholder만.
- **글자색** — 흰색(기본)·강조(amber)·빨강·파랑·초록 스와치 + 🎨 직접 선택. 흰색으로 되돌리기 가능.
- **일괄 입력** — 상단 바의 **⤵ 대본 일괄 입력**(PDF 교체 버튼 옆)에서 전체 대본을 한 번에 붙여넣기. 페이지 구분 기호 `=== 1 ===` 아래에 그 페이지 대본. (`--- 페이지 2 ---`, `# 3` 도 인식. 강조 `**별표**`)
- **LLM 프롬프트** — 일괄 입력 창의 📋 버튼으로 장수·시간·형식 규칙이 담긴 프롬프트를 복사 → LLM에 주고 받은 결과를 그대로 붙여넣기.
- **내보내기/가져오기** — 대본·시간·영상 설정을 JSON으로 백업/이동.

### 2. 총 발표 시간 · 페이지별 권장시간 (발표자 설정)
**총 발표 시간**은 시작 화면과 타이머 영역의 **총 발표 시간 [N] 분**에서 설정(저장됨, 카운트다운 기준). 페이지별 권장시간은 사이드바 **이 페이지 권장 [N] 초**에서 페이지마다 직접 설정·저장. 기본값은 총 발표시간÷장수. 일괄 입력에서 `=== 3 (60s) ===`로도 지정 가능. 큰 타이머에 남은 시간·페이지 경과·총 경과 표시.

### 3. 페이지에 영상 삽입 (유튜브 / 로컬 mp4)
🎬 영상 첨부로 페이지에 **유튜브 링크** 또는 **로컬 mp4**를 붙인다. 발표 중 **▶ 영상(또는 `V`키, 리모컨 버튼)** → 연단 슬라이드 위에 전체화면으로 재생, 페이지 이동·`Esc`로 닫으면 슬라이드 복귀.
- 유튜브: 모든 기기(크로스 디바이스)에서 재생.
- 로컬 mp4: 이 브라우저에 저장 → **같은 PC deck**에서 재생(다른 기기 deck은 유튜브 권장).

### 4. 슬라이드 띄우기 & 동기화
- **같은 노트북 2화면** — ‘프로젝터에 슬라이드 띄우기’ → deck 창이 코드 없이 즉시 따라옴(BroadcastChannel, 네트워크 불필요). 확장 디스플레이면 프로젝터에 자동 배치.
- **다른 기기 연단** — `deck.html` 열고 3자리 코드 입력(PeerJS/WebRTC). 매 세션 랜덤 3자리라 학교망 충돌 회피.
- **폰 리모컨** — QR 스캔 → PREV/NEXT/가림/처음/영상 + 볼륨·방향키(블루투스 클리커). 페이지·타이머·대본 미리보기 표시.

## 조작 키 (프롬프터)
`→/Space` 다음 · `←` 이전 · `+/−` 글자 크기 · `E` 편집 · `V` 영상 · `Esc` 영상 닫기/가림 · `Home/End` 처음/끝 · 대본 영역 클릭 좌/우.

## 파일 구조
```
prompter.html   호스트(컨트롤)        index.html  → prompter.html 리다이렉트
deck.html       연단 슬라이드(뷰어)    .nojekyll   GitHub Pages 설정
remote.html     폰 리모컨
js/  config · pdf-render · sync · notes
css/ app.css
docs/ 스크린샷
```

## 다른 발표에 재사용
`js/config.js`의 `appTitle`·`targetSec` 정도만 바꾸면 된다. PDF는 시작 화면에서 불러오고, 새 PDF를 불러오면 그 파일명 기준으로 대본이 분리 저장된다.

## 연결이 안 될 때
- 세 기기를 **같은 핫스팟**에 둔다(가장 확실). · HTTPS(Pages) 사용 시 WebRTC가 안정적. · 같은 노트북 2화면은 네트워크가 전혀 필요 없다.
